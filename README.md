## 提交格式

* 提交类型
  * feat: 新功能
  * fix: 修复问题(bug)
  * docs: 修改文档
  * style: 修改代码格式，不影响代码逻辑
  * refactor: 重构代码，理论上不影响现有功能
  * perf: 提升性能
  * test: 增加修改测试用例
  * chore: 修改工具相关（包括但不限于文档、代码生成等）添加依赖 修改格式等
  * deps: 升级依赖
  * revert: 回滚版本
  * build: 构建系统或者包依赖更新
* 范围Scope
    * 用于说明本次修改涉及的范围-模块
* 主题 Subject
* 详细说明 Body

* 例如:
```
<type>(<scope>): <subject>
<空行>
<body>
<空行>
<footer>
**************************************
feat(模块): 新增功能
-- 空行
详细说明
```
